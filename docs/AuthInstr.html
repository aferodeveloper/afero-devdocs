Developer Doc Authoring Instructions

When writing in the developer docs, please use the instructions and HTML tags and classes included in this document. The developer docs are in the “devportal” git repo. The posted HTML files are in the devportal://src/client-side/views/ directory. Linked files and image files are in the devportal://src/client-side/static/custom/files and /images directories, respectively.

Authoring Environment

To get started with the developer docs, you must first get your environment ready by doing the following:

  1. Install Node.js.
  2. Log in to npm using the command line interface.
  3. If you don’t have an npm account, create one. Then see someone on the service team to get your account added to our corporate account.

Once your environment is set up, do the following:

  1. From the command line, go to the base directory of the devportal repo. You’ll know you’re in the right place if, when you type you ls, you should see a package.json file.
  2. Install all the necessary libraries by typing: npm install
  3. Start the server by typing: npm run local
  4. Point your browser to http://localhost:3000/ to view the documentation set.

How to Add a New Developer Doc File

When writing new content, follow these steps:

  1. Make a copy of /src/client-side/views/Template.html, then rename it using a short name that clearly and uniquely identifies the content. Use initial caps to separate words; e.g., Modulo2DataSheet.html. If you must use a dash, that’s ok, but don’t use underscores or spaces in the filename.

  2. Open this file for editing: /src/server-side/server.js. You must add your new file to the list of dev doc HTML files. It doesn’t matter where you add it in the list of files, but try to insert it in a logical place. The format to use is:

    {
 file: "SystemOverview",
 pageDescription: "",
 pageTitle: "Afero IoT Platform Overview | Afero",
 twitterPageTitle: "Afero IoT Platform Overview",
 next: {
   id: "025-237C3t5",
   href: "/CoreConcepts",
   title: "Core Development Concepts"
 }
},

    Where:

    file HTML filename, without the “.html”.
    pageDescription Leave blank, for now.
    pageTitle The title of the page (H1 tag contents). Add the “| Afero” as shown in the example above.
    twitterPageTitle The title of the page (H1 tag contents).
    next* id - Create using the ider.py script, described in Links, Step 2.
    href - The filename of the next page in the section, without the .html extension.
    title - The page’s H1 title content.

    *Within each major section of the developer docs, we provide navigation at the bottom of the pages to the next file in that section. You define this navigation in the next attributes described above. Note that even if there is no file that comes next in the section, include a next entry, but do not put any information within the brackets; e.g.:

    {
file: "LabX",
pageDescription: "",
pageTitle: "Lab X: Atmel XPlained Pro Demo | Afero",
twitterPageTitle: "Lab X: Atmel XPlained Pro Demo",
next: {}
},

    Pay close attention to proper punctuation in the server.js file!

    After modifying the server.js file, you must restart npm (by typing npm run local) in order to view the new file.


  3. Make sure you can view your new file by starting up npm and pointing your browser to “localhost:3000/NewFileName”. It’s not uncommon for npm to fail to start due to some syntax error in the server.js file. If this happens, check your new entry in server.js and try again.
  4. Create your new document content following HTML markup conventions listed below in HTML Instructions. Be careful to retain all the content that you started with (from Template.html), except the commented-out portion. The file clearly identifies where you should substitute your new content: <!-- ########### INSERT CONTENT HERE ########### -->.
  5. When your new content has been reviewed and is final, add your page into the left-hand Navigation pane:
    1. Open the file: /src/client-side/views/partials/nav-data.html.
    2. Locate where you want to insert the new page in the list of files and add your file, following the syntax used. For example: 
      {
	link: "/Overview",
	name: "Introducing " + locale.CompNameShort,
	id: "TOC-Intro",
	subMenu: [
	{
		link: "/SystemOverview",
		name: locale.CompNameShort + " IoT Platform Overview",
		id: "TOC-Overview"
	},
	{
		link: "/CoreConcepts",
		name: "Core Development Concepts",
		id: "TOC-CoreConcepts"
	}]
},

      You can create two levels of submenus, as shown here:

      {
	link: "/FW-API",
	name: "Firmware Reference",
	id: "TOC-FW-API",
	subMenu: [
	{
		link: "/MCUtoHachi",
		name: "MCU to ASR Communication",
		id: "TOC-MCUtoHachi",
		subMenu: [
		{
			link: "/afPro-SPI",
			name: "afPro SPI Protocol - How It Works",
			id: "TOC-afPro-SPI"
		},
		{
			link: "/afPro-UART",
			name: "afPro UART Protocol - How It Works",
			id: "TOC-afPro-UART"
		}]
	},
...
},
  6. Check that your new page shows up correctly in the Navigation pane; make adjustments to the nav-data.html file until the page displays correctly.

HTML Instructions

We currently follow the HTML5 standard.

Characters and Other Conventions

  • Indent the HTML code using tabs, not spaces.
  • Use “curly quotes” and ‘curly apostrophes’.
    Note! If you set your HTML/text editor to automatically use the curly quotes, then when you give a tag class a value, you must do whatever it takes to use straight quotes, or the value won’t be recognized.
  • Use the ellipsis character: … Never use three periods for an ellipsis as they can break across lines.
  • Spaces (&nbsp;) should not be used for formatting purposes - instead, use CSS margin styling.
  • Always place punctuation outside quote marks; for example, in the fragment below, the comma is placed outside the ending quote after Mindy:
    …if your ClientID was “Mork” and your Client Secret was “Mindy”, you would…
  • Spell out numbers zero through nine, unless a higher number appears in the same sentence.

Variables

Variables are defined in this file: devportal://locale/en/locale.json.

To reference these terms in the HTML files, use the format: <%- locale.variable-name %>. For example, this code: <%- locale.MobileAppLong %> will display as: <%- locale.MobileAppLong %>.

Headings

Begin each HTML file with an <h1> tag. If the heading maps to a top-level heading in the Navigation pane, add this class: <h1 class="main-head">. This will create an all-caps heading.

For all headings, add an id if you think you might want to link to it. For example:

<h2 id="First-Heading">My First Heading</h2>. Use hyphens when naming IDs to separate words.

Lists

Use ordered lists for steps or for displaying information that occurs sequentially. Use unordered lists for plain lists that imply no order. It’s common sense.

Keep ordered lists to no more than two levels, using numbers then lowercase alpha characters. If you find you need a third level, try hard to reorganize the information so it doesn’t require three levels. There are some exceptions to this, especially for hardware checklists, such as in the <%- locale.OurChip1 %> Hardware Integration Guide.

Unordered Lists

  • For your basic, bulleted list, use <ul class="af-ul">.
  • To create a list without markers, use <ul class="af-ul no-bullet">.
    • Level 2 unordered list. Use <ul class="af-ul-2">.
    • Another level 2 item.
      • Level 3 nested list. Use <ul class="af-ul-3">.
      • Use this many levels only in an emergency!

Ordered Lists

  1. Use <ol class="af-ul">.
  2. Another list item.
    1. Level 2 ordered list using <ol class="af-ol-lower-alpha">.
    2. Another list item.
  3. Final item in this ordered list.

Links

Every link (anchor tag) must contain an ID that is unique across the developer docs. This is for auto-test purposes. We use a timestamp as the ID and generate it using a script. The script, ider.py, is located in the same directory as the HTML files (/src/client-side/views/). Note that you might have to give yourself execute permission first by typing: chmod +x ider.py

“Next” page links are special. Most of the links you’ll create will be within the HTML pages that make up the developer portal. The exception to this is the “Next” page link at the bottom of pages within sections. These links are defined in the /src/server-side/server.js file, and you generate their IDs using the Template.html file. Read comment 12 in Template.html for details and use the ider.py directions in step 2 below.

To create an ID for a link in an HTML file living in /src/client-side/views/, do the following:

  1. In your HTML file, within the <a> tag, create the ID as follows: <a id="" href="/File- or anchor-name">. The script will look for occurrences of id="" (single quotes also work), and then will insert a timestamp inside the quotes.

  2. Run the ider.py script in the HTML working directory:

    Usage:

    ider.py [-h] [-d] target_file

    Positional arguments:

    target_file     Path to the file to be ID’d.

    Optional arguments:

    -h, --help      Show this help message and exit.

    -d, --dry_run   Don’t save; just report what it WOULD do.

    As you can see, you can provide an optional -d to make it just say what it would do, but not save the output. The script will print the file (with inserted IDs) to your console, and then report how many changes it would have made.

    If you don’t provide -d, the script will overwrite the input file and report how many changes it made.

Span Tags

Within paragraphs, you can use <span> tags to create inline styles. We’ve defined the following span tags:

Span Class Description
internal Text is large and red for visibility. Use to note to-do items, questions, etc. These must be removed before a deployment so use with care.
UIText Text is bold and blue. Use to call out UI elements, such as button names, menu item names, and field labels.
highlight-orange No real purpose but it exists.

For bold and italic, use the <strong> and <em> tags, respectively.

Images

Source your image files (screenshots, diagrams, etc.) using Sketch or Adobe Photoshop if possible. You can place related images in the same source file, then export slices. Try to name the source file and the slice files using the same base so they can be related if updates to the slices are needed.

For <img> tags:

  • The attributes src and alt are required.
  • If you need to size the image, only specify the height attribute, in pixels (default).
  • Use class="img-inline" for small images inline with text, such as illustrative icons.
  • Use class="img-br" if within <ol> or <ul> tags and if image is not inline.

You can also use the parental <figure> element, which can include a <figcaption> tag in addition to the required <img> tag. We don’t usually use captions with figures, but if you do, position it above (or below) the image by placing the <figcaption> tag before (or after) the <img> tag. For example:
